> 本文件是 My Prompt Maker 指令產生器的核心參考架構，由 Anthropic、Google Gemini API、OpenAI/ChatGPT 三大官方指南提煉而來。供 Meta-Prompt 引用。

- `提示设计策略 _ Gemini API.md`（Google Gemini API 官方中文指南）

## 一、通用 7 層結構架構

三大 AI 廠商在提示詞設計上有高度共識，以下 7 層架構是最大公約數：

| 層次 | 名稱 | 英文 | 定義 | 對應框架 |
|------|------|------|------|---------|
| 1 | **角色層** | Role / Persona | 設定 AI 扮演的角色與知識背景，啟動領域模式 | Claude: system prompt role；Gemini: role 標籤；ChatGPT: POWER-R（Refinement 角色段） |
| 2 | **目標層** | Task / Purpose | 明確說明要達成什麼，避免模糊指令 | POWER-P（Purpose）；Gemini 任務輸入型 |
| 3 | **上下文層** | Context | 提供背景資訊、受眾定義、使用情境、資料來源 | POWER-W（Working Context）；Gemini 上下文添加 |
| 4 | **示例層** | Examples / Few-shot | 2–5 個具體示範，引導格式與風格 | Claude `<examples>` 標籤；Gemini 少樣本提示；POWER-E（Examples） |
| 5 | **輸出規格層** | Output Format | 格式（YAML/JSON/MD）、長度、結構、呈現方式 | POWER-O（Output Format）；Gemini 回答格式指令 |
| 6 | **約束條件層** | Constraints | 禁止事項、必要條件、品質標準、安全邊界 | Claude `<constraints>`；Gemini 限制條件；POWER-R（Refinement 限制段） |
| 7 | **精煉層** | Refinement | 語氣、風格、品質標準、自我審查指令 | POWER-R（Refinement）；Claude 迭代精煉；Gemini 提示迭代策略 |

## 二、三大廠商特殊語法規範

### Claude（Anthropic）

**核心特性：XML 標籤結構化**

```xml
<instructions>
  你是一位專業的視覺設計師，專精於簡報底圖設計。
</instructions>

<context>
  使用場景：NotebookLM 知識型簡報，觀眾為商業受眾
</context>

<examples>
  <example>
    輸入：科技未來感
    輸出：深邃黑底搭配霓虹青漸層，融入數據網格紋理...
  </example>
</examples>

<task>
  請依照以下分類架構，為我生成多風格簡報背景提示詞。
</task>
```

**關鍵規則：**
- 將指令、上下文、示例、任務分別放入不同 XML 標籤
- 長文件使用 `<document>` 標籤包裝
- 角色設定在 system prompt 而非 user message
- 正面說明（「請做 X」）優於負面說明（「不要做 Y」）

---

### Google Gemini

**核心特性：結構化 XML/Markdown + 指令放末端**

```xml
<role>
你是一位專業的視覺設計師，專精於 AI 生成藝術與簡報底圖設計。
</role>

<constraints>
1. 所有提示詞必須符合 16:9 橫式比例
2. 背景不得干擾前景文字閱讀
</constraints>

<context>
[在此放入大量背景資料或文件]
</context>

<task>
依照色彩系統、材質紋理、光線情境三大分類，各生成一個簡報背景提示詞。
</task>
```

**關鍵規則：**
- **指令放最末端**（長上下文時，指令在末尾效果提升 30%）
- 大量上下文先放，具體指令後放
- 使用清晰分隔符區分提示各部分
- 支援 `thinking_level: "high"` 啟動深度推理（API 參數）
- 少樣本示例：建議 3–5 個，格式必須一致

---

### ChatGPT / OpenAI

**核心特性：POWER 框架 + CO-STAR 框架**

**POWER 框架範例：**
```
[P - Purpose]
為 NotebookLM 知識型簡報生成高品質背景底圖提示詞

[O - Output Format]
每個提示詞同時輸出：YAML 格式（依規範）+ 自然語言格式（100-200字）

[W - Working Context]
使用場景：B2B 知識分享簡報，受眾為商業決策者
限制：16:9 比例，8K 解析度，背景不干擾前景

[E - Examples]
參考範例：「生成一個高品質的抽象背景簡報，使用深邃黑和霓虹青色漸層...」

[R - Refinement]
風格：前衛 AI 科技美學；精緻度對齊專業設計師水準；避免過於商業化的通用感
```

**CO-STAR 框架（適合創意類任務）：**
```
Context（背景）: NotebookLM 知識型簡報製作
Objective（目標）: 生成多風格背景底圖提示詞
Style（風格）: 前衛科技美學
Tone（語氣）: 專業視覺設計師語彙
Audience（受眾）: 使用 AI 圖片生成工具的設計師或 PM
Response（回應）: YAML + 自然語言雙格式輸出
```

**關鍵規則：**
- 指令清晰具體，避免「讓它更好」這類模糊描述
- 避免對新一代推理模型（o-series）手動加 Chain-of-Thought
- 使用 POWER 或 CO-STAR 任一框架，不要混用

## 三、提示詞技術類型

| 技術 | 適用情境 | 寫法特點 |
|------|---------|---------|
| **Zero-shot** | 任務清晰、格式標準 | 直接描述目標，無需示例 |
| **Few-shot** | 需要特定格式或風格 | 提供 2–5 個「輸入 → 輸出」示例對 |
| **Chain-of-Thought（CoT）** | 數學、規劃、複雜推理 | 加入「請先說明你的推理步驟，再給出答案」 |
| **RAG（資料引用）** | 答案需基於特定文件 | 貼入資料，加「僅使用以下資料作答」 |
| **Metacognitive（元認知）** | 需要高品質分析 | 讓 AI 先列出假設與限制，再回答 |
| **Iterative（迭代精煉）** | 需要高品質創作 | 「先起草 → 自我評審 → 修改輸出」三步驟 |
| **Prompt Chaining（串接）** | 多步驟複雜任務 | 每個 API 呼叫只做一件事，輸出作為下一個輸入 |

## 四、視覺提示詞專屬分類架構（A–G）

本分類架構專為**圖片生成、簡報底圖、視覺設計**場景設計，對應 NotebookLM 示範用例。

| 分類 | 名稱 | 核心問題 | 設計維度 |
|------|------|---------|---------|
| **A** | **色彩系統** | 用什麼顏色？如何搭配？ | 主色調、輔色、漸層方向、色彩情感 |
| **B** | **材質紋理** | 表面有什麼視覺元素？ | 紋理類型、透明度、比例尺度、疊加方式 |
| **C** | **光線情境** | 光線如何打造氛圍？ | 光源類型、光線效果、光影對比、情緒氛圍 |
| **D** | **構圖版式** | 視覺重量如何分配？ | 焦點位置、留白比例、動態感、層次深度 |
| **E** | **設計美學** | 整體屬於哪種美學風格？ | 設計流派、時代感、視覺語言、文化參照 |
| **F** | **技術規格** | 輸出的技術參數是什麼？ | 解析度、比例、色彩空間、AI 生成參數 |
| **G** | **功能性需求** | 如何確保設計可用性？ | 前景相容性、安全區域、對比度、可讀性 |

### 各分類詳細設計維度

**A - 色彩系統**
- 主色調：單色、雙色、三色、漸層
- 色彩情感：冷色系（科技/冷靜）、暖色系（活力/溫暖）、中性（專業）
- 配色策略：互補色（高對比）、類比色（和諧）、分裂互補（動感）
- 漸層方向：線性（水平/垂直/斜角）、放射狀、角度

**B - 材質紋理**
- 幾何型：晶格、網格、電路板、像素、折紙
- 有機型：大理石、紙張、布料、水面、木紋
- 科技型：數據流、代碼雨、全息投影、粒子場
- 透明度層次：微隱（5-15%）、半透明（30-50%）、明顯（70%+）

**C - 光線情境**
- 光源：環境光、定向光、霓虹燈光、全息投影光
- 效果：光暈（Glow）、光線透射（Ray）、反射（Reflection）、焦散（Caustics）
- 氛圍：神秘（低調暗光）、科技（霓虹冷光）、溫暖（暖色漫射）、戲劇（高對比聚光）

**D - 構圖版式**
- 焦點位置：置中、三分法、黃金比例、對角線
- 留白策略：左側（文字置右）、右側（文字置左）、下方（標題在上）
- 深度感：前中後三層次、透視消失點、縮放重疊

**E - 設計美學**
- 科技系：賽博龐克（Cyberpunk）、新未來主義（Neo-Futurism）、科技極簡（Tech Minimalism）
- 商業系：企業正式（Corporate）、現代簡約（Modern Clean）、北歐極簡（Scandinavian）
- 藝術系：包豪斯（Bauhaus）、孟菲斯（Memphis）、日本美學（Japandi）、復古未來（Retrofuturism）

**F - 技術規格**
- 解析度：1080p（FHD）、4K（UHD）、8K（超高清）
- 比例：16:9（橫式簡報標準）、4:3（傳統簡報）、1:1（方形）、9:16（直式）
- Midjourney 參數：`--ar 16:9 --style raw --v 6.1 --quality 2`
- 色彩空間：sRGB（螢幕標準）、DCI-P3（高色域）

**G - 功能性需求**
- 前景文字相容性：文字顏色對比度 ≥ 4.5:1（WCAG AA 標準）
- 安全區域：邊緣留 5-10% 不放關鍵元素
- 視覺干擾控制：背景複雜度不蓋過前景資訊層

## 五、模型選用建議

| 任務類型 | 建議模型 | 理由 |
|---------|---------|------|
| 高品質創意生成（簡報提示詞） | Claude Opus 4.6 | 視覺美學理解強，XML 結構化效果佳 |
| 快速批量生成 | Claude Sonnet 4.6 或 Gemini 2.5 Flash | 速度與品質平衡 |
| 視覺圖片實際生成 | Midjourney v6.1 | 最佳藝術品質 |
| 商業簡報配圖生成 | DALL-E 3（透過 ChatGPT）| 文字與圖片整合度佳 |
